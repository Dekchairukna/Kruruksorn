# อัปเดต: จัดระบบเนื้อหาให้เป็นทางเดียว

แก้โครงสร้างการจัดการเนื้อหาให้ชัดเจนเป็นลำดับเดียว:

รายวิชา → หน่วยการเรียนรู้ → บทเรียน → ใบงาน/แบบทดสอบ

## สิ่งที่แก้

1. หน้า `subject_detail.html` แตกบทเรียนย่อยใต้แต่ละหน่วยแล้ว
   - เห็นทันทีว่าแต่ละบทเรียนมีใบงานกี่ใบ แบบทดสอบกี่ชุด และไฟล์แนบกี่ไฟล์
   - ปุ่ม `+ ใบงานบทนี้` และ `+ แบบทดสอบบทนี้` อยู่ในแถวของบทเรียนโดยตรง
   - ลดปัญหาเพิ่มใบงานในหน่วยหนึ่ง แต่ไปเปิดอีกบทเรียนแล้วนักเรียนไม่เห็น

2. หน้าเปิดคาบเรียน `schedule_period.html`
   - รายการเลือกบทเรียนแสดงชื่อหน่วย + ชื่อบทเรียน + จำนวนใบงาน/แบบทดสอบ/ไฟล์
   - ทำให้ครูรู้ก่อนกดเปิดว่า บทเรียนที่เลือกมีใบงานหรือยัง

3. ฝั่ง route `subject_detail`
   - เปลี่ยนจากสรุปจำนวนรวมอย่างเดียว เป็นส่งข้อมูลแบบ nested lesson rows
   - เรียงหน่วยและบทเรียนด้วย id ชัดเจน

4. ลบบทเรียนปลอดภัยขึ้น
   - ลบบทเรียนพร้อมข้อมูลลูก เช่น ใบงาน ข้อคำถาม คำตอบ แบบทดสอบ ไฟล์แนบ และงานประจำคาบที่ผูกอยู่

## หลักการใช้งานหลังแก้

- ถ้าจะให้ใบงานขึ้นในคาบเรียน ต้องเพิ่มใบงานที่ `บทเรียน` ที่ครูจะเปิดในคาบนั้น
- ไม่เพิ่มใบงานลอย ๆ ที่ระดับหน่วยอีก
- หน้าโครงสร้างรายวิชาจะแสดงให้เห็นชัดเจนว่าใบงานอยู่บทเรียนใด
